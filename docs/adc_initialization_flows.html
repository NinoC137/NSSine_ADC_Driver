NS800RT7P65 ADC 设备初始化流程

本图把 ADC 使用拆成统一的开发者流程：配置 ADC unit 基础功能，配置 ADCIN 与 SOC，按需要配置 IRQ、DMA、Burst、PPB 等功能，最后启动采样并在回调或读取路径中处理结果。 工程中的基础抽象位于 ns_adc_unit、ns_adc_irq、ns_adc_dma 与 PPB 模块，具体 demo 流程放在 applications/adc_example.c 和 applications/adc_thread.c。

ADC unit 基础功能 Channel/SOC 配置 IRQ/回调 DMA/DMAMUX/EDMA Burst/触发源 PPB 后处理 启动与结果处理

抽象层边界

应用/demo

选择 ADCA/B/C、ADCIN、SOC、功能组合；定义 IRQ 回调和结果处理策略。

unit 抽象

ns_adc_get、ns_adc_slot、ns_adc_trigger、ns_adc_read。

IRQ 薄封装

ns_adc_irq_on/off、attach/detach，只负责 INT1~4 与 NVIC 分发。

显式模块层

ns_adc_dma、Burst 配置和 PPB 模块按需组合，应用直接选择需要的能力。

硬件/SDK

ADCA/B/C、32 个 SOC、ADCIN0~31、PPB1~4、DMAMUX/EDMA、ePWM。

统一初始化模板

  • 先选择 ADC unit：ADCA、ADCB 或 ADCC。
  • 配置 ADC 基础功能：参考电压、分频、采样脉冲位置、converter enable。
  • 配置 ADCINx -> SOCn，SOC 是 ADC 内部采样任务槽位。
  • 按功能选择 IRQ、DMA、Burst、PPB；不用的功能不要隐式开启。
  • 启动采样后，结果处理可以是轮询读取、IRQ 回调、DMA 完成回调或 PPB 状态读取。

基础单次采样

一次读取一个 ADCIN。适合调试、低速轮询和 RT-Thread 标准 rt_adc_read() 路径。

配置 ADC unit 基础功能

ns_adc_hw_init() 初始化 VREF、prescaler、pulse position，并启用 converter。

配置 channel 与 SOC

ns_adc_channel_to_slot() 得到 ADCINx、SOCn，再调用 ns_adc_slot()。

配置 EOC 标志

ns_adc_irq_on(unit, INT1, soc)，把目标 SOC 的 EOC 绑定到 ADCINT1。

软件触发一次采样

ns_adc_trigger(unit, ns_adc_mask(soc))，只触发当前 SOC。

等待并读取结果

ns_adc_wait() 等待 INT flag，随后 ns_adc_read(unit, soc) 读取 raw。

连续采样 IRQ

开发者显式配置多个 SOC 与 INT1~4，对每次 EOC 进入应用层回调处理。

配置 ADC 基础功能

初始化 ADCA/B/C。demo 中选择 ADCA。

配置多组 ADCIN/SOC

例如 ADCIN0~3 -> SOC0~3，逐个调用 ns_adc_slot()。

注册回调函数

ns_adc_irq_attach(unit, callback)，回调在 demo 中读取 SOC、清 flag、释放同步对象。

配置 INT1~4

例如 SOC0 -> INT1、SOC1 -> INT2，调用 ns_adc_irq_on()。

循环触发采样

按需要触发单个 SOC 或 SOC mask，回调中保存结果。

退出清理

ns_adc_irq_off() 关闭 INT，ns_adc_irq_detach() 注销回调，避免影响轮询采样。

ADC Burst

由触发源启动一组连续 SOC，适合固定序列快速采样。当前 demo 默认用 ePWM SOCA 触发。

配置 ADC 基础功能

选择 unit，并确保 converter、VREF、prescaler 已配置。

配置 Burst SOC 序列

从 first_soc 开始，把 adc_channels[] 展开到多个 SOC。

配置 Burst 功能

ns_adc_burst() 设置 burst size、priority 和 Burst trigger。

配置外部触发源

例如配置 ePWM1 SOCA 的 period、compare、prescale，并打开 SYSCON 路由。

绑定最后一个 SOC 的 EOC

将最后一个 SOC 绑定到 ADCINT1，用于判断整组 burst 已完成。

等待完成并读取序列

ns_adc_wait() 后读取 first_soc ... first_soc+N-1 的结果。

连续采样 + DMA

ADC EOC 产生 DMA 请求，EDMA 将结果搬运到内存缓冲区，CPU 只处理 DMA 完成中断。

配置 ADC 基础功能

初始化 unit，并配置需要采样的 SOC。

配置采样 SOC

选择一个或多个 SOC，设置 trigger、ADCIN、sample window。

配置 DMAMUX

把 ADC DMA 请求源，例如 DMAMUX_ADCA1_REQ，路由到 EDMA channel。

配置 EDMA TCD

源地址是 ADC result 寄存器，目标地址是对齐 buffer，设置 minor/major loop。

注册 DMA 完成中断

EDMA major loop 完成后进入 EDMA1_CH0_IRQHandler，失效 DCache 并汇总结果。

启动 ADC 与 DMA

启动 EDMA，再由软件或外部触发持续产生 ADC 结果。

ADC Burst + DMA

Burst 负责批量产生 SOC 结果，DMA 负责把结果序列搬到内存，适合固定窗口高速采样。

配置 ADC 基础功能

选择 unit，完成 converter、VREF、时钟分频和结果寄存器路径配置。

配置 Burst SOC 序列

配置 first_soc、burst_size 和 adc_channels[]。

配置 Burst 触发

打开 Burst mode，配置 ePWM 或其他 Burst trigger。

配置 DMA 搬运

DMAMUX 选择 ADC 请求源，EDMA buffer 按结果宽度和 sample count 配置。

配置完成中断

可选择 DMA major interrupt 或最后 SOC 的 ADCINT 作为完成信号。

启动触发链路

启动 EDMA，再启动 ePWM/Burst 触发，完成后从 DMA buffer 读取窗口数据。

ADC PPB 后处理

PPB 挂在某个 SOC 后面，可做限值事件、delay timestamp、过采样 SUM/COUNT/MIN/MAX。

配置 ADC 基础功能

初始化 unit，并准备 PPB 要绑定的 SOC。

配置 PPB 对应 SOC

通过 ADC_setupPPB(unit, ppb, soc) 或封装层绑定 PPB1~4 与 SOC。

选择 PPB 功能

limit 模式配置 high/low/event；oversampling 配置 limit、shift、sync。

按需注册 Event IRQ

PPB trip、zero 或 OSINT 可进入 ADCA_EVENT_IRQHandler 一类事件入口。

启动采样或同步事件

通过 SOC 触发、外部同步或软件 sync 推进 PPB 后处理逻辑。

读取 PPB 状态

读取 event flags、delay stamp、SUM、COUNT、MIN、MAX、平均值或剔除极值平均值。

设计原则：驱动层提供可复用的 ADC unit、IRQ、DMA、Burst、PPB 配置接口；demo 层负责选择功能组合、定义回调和决定结果处理方式。 因此 IRQ demo 的回调函数属于应用层，而 ns_adc_irq.c 只负责把 ADCA/B/C 的硬件入口分发到用户注册的回调。

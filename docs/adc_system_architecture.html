NS800RT7P65 RT-Thread ADC Demo 系统架构

当前工程采用“ADC unit + 显式资源配置 + 标准设备”结构：ns_adc_unit 统一封装 ADCA/ADCB/ADCC；adc0 保持官方 RT-Thread ADC API 兼容并默认暴露 ADCIN0~31；DMA、Burst、PPB 和 IRQ 由独立模块按需组合。

硬件抽象模型

3 个 ADC unit

ADCA、ADCB、ADCC 是三个独立 ADC 外设单元，每个 unit 有自己的 base、result、IRQ 和 DMA 请求。

32 个 SOC / unit

每个 unit 内部有 SOC0~SOC31。SOC 是转换任务槽，用于绑定触发源、ADCIN 和采样窗口。

4 个 PPB / unit

每个 unit 内部有 PPB1~PPB4，可显式绑定到某个 SOC 做 limit/event/delay 后处理。

ADC Unit 抽象层

ns_adc_unit

一个结构体代表一个硬件 ADC 单元，保存 base/result、CONV/EVENT IRQ、DMAMUX 请求源和默认采样参数。

短函数接口

ns_adc_slot/seq/once/wait 负责 SOC 序列和单次采样；irq/dma/burst/ppb 作为独立模块按需组合。

设备层复用

adc0 与应用示例都不直接散落寄存器地址；统一通过 unit/slot/seq 抽象访问 ADCA/ADCB/ADCC。

PPB 独立模块

drv_adc_ns800rt7p65x_ppb.c

集中处理 PPB1~PPB4 绑定、limit high/low 配置、EVENT IRQ 挂接和 delay timestamp 读取。

drv_adc_ns800rt7p65x_ppb.h

事件路径

EVENT IRQ 进入 ADCA/B/C_EVENT_IRQHandler 后，按当前 unit 的 PPB 配置读取并清除事件状态。

应用层 RT-Thread BSP 适配 厂商 SDK 硬件外设

ADC 外设如何挂载到 RT-Thread

1

INIT_DEVICE_EXPORT(rt_hw_ns_adc_init) 在 RT-Thread 组件初始化阶段执行，调用 rt_hw_adc_register() 注册标准设备 adc0。

2

官方 ADC core 保存 struct rt_adc_ops。应用调用 rt_adc_read() 时，最终进入本 BSP 的 ns_adc_convert()。

3

ns_adc_convert() 将 RT channel 转为 ns_adc_slot_cfg，调用 ns_adc_once() 只触发目标 SOC，并在超时保护内读取结果。

4

DMA、Burst、PPB 等能力不再通过额外设备注册；应用或调试命令直接调用独立 BSP 模块，避免把自定义控制命令塞进官方 rt_adc_ops。

中断如何进入 RT-Thread 驱动

启动文件的向量表已经包含 ADCA_CONV_IRQHandler、ADCC_CONV_IRQHandler、ADCA_EVENT_IRQHandler、EDMA1_CH0_IRQHandler 等弱符号。本工程在 ns_adc_irq.c 中定义 ADC CONV/EVENT handler，链接时自然覆盖默认 handler。

运行时，IRQ 封装可调用 Interrupt_register(IRQn, handler)。SDK 里的这个函数本质是 NVIC_SetVector()，所以它只是把单个 IRQ 的入口改到指定 handler；随后通过 NVIC_EnableIRQ() 使能中断。

本工程不要调用 SDK 示例里的 Interrupt_initModule() 或 Interrupt_initVectorTable() 作为常规初始化，因为 RT-Thread 启动流程已经建立异常/中断上下文。重新初始化整张向量表可能覆盖内核使用的异常入口。

当前关键约束

  • adc0 保持官方 RT-Thread ADC API 兼容，不承载自定义高级命令。
  • adc0 默认暴露 32 路，映射为 ADCA SOCN → ADCINN；这只是 demo 默认策略。
  • DMA、Burst、PPB 作为独立模块被应用显式调用，不再额外注册伴生设备。
  • SDK 与 RT-Thread 官方源码保持只读；适配逻辑只放在本 BSP 工程内。
  • Burst 示例默认使用 ADCA + EPWM1 SOCA；DMA 示例默认走 EDMA1 CH0 与对应 DMAMUX ADC 请求源。

应用层 RT-Thread BSP 适配 厂商 SDK 硬件外设

ADC 外设如何挂载到 RT-Thread

1

INIT_DEVICE_EXPORT(rt_hw_ns_adc_init) 在 RT-Thread 组件初始化阶段执行，调用 rt_hw_adc_register() 注册标准设备 adc0。

2

官方 ADC core 保存 struct rt_adc_ops。应用调用 rt_adc_read() 时，最终进入本 BSP 的 ns_adc_convert()。

3

ns_adc_convert() 根据通道映射取得 ns_adc_unit 与 SOC，调用 ns_adc_trigger() 软件触发转换，再通过 ns_adc_read() 读取结果。

4

INIT_DEVICE_EXPORT(rt_hw_ns_adc_adv_init) 注册高级伴生设备 nsadc0，它不是 ADC class，而是 Misc device，避免污染官方 rt_adc_ops。

中断如何进入 RT-Thread 驱动

启动文件的向量表已经包含 ADCA_CONV_IRQHandler、ADCC_CONV_IRQHandler、ADCA_EVENT_IRQHandler、EDMA1_CH0_IRQHandler 等弱符号。本工程在 drv_adc_ns800rt7p65x_adv.c 中定义同名强符号，链接时自然覆盖默认 handler。

运行时，高级驱动还会调用 Interrupt_register(IRQn, handler)。SDK 里的这个函数本质是 NVIC_SetVector()，所以它只是把单个 IRQ 的入口改到指定 handler；随后通过 NVIC_EnableIRQ() 使能中断。

本工程不要调用 SDK 示例里的 Interrupt_initModule() 或 Interrupt_initVectorTable() 作为常规初始化，因为 RT-Thread 启动流程已经建立异常/中断上下文。重新初始化整张向量表可能覆盖内核使用的异常入口。

标准路径与高级路径

标准采样
adc0 → rt_adc_ops → ns_adc_convert → ns_adc_unit → SDK ADC
高级采样
nsadc0 → rt_device_control/read → ns_adc_unit → IRQ/DMA/Burst/PPB
完成通知
ISR 写入 sample buffer，并释放 rt_sem 或触发 rx_indicate
配置入口
Kconfig 控制高级功能，SConscript 按需加入 SDK 源文件

当前关键约束

  • adc0 保持官方 RT-Thread ADC API 兼容，不承载自定义高级命令。
  • nsadc0 是高级能力扩展点，第一版高级模式互斥运行。
  • SDK 与 RT-Thread 官方源码保持只读；适配逻辑只放在本 BSP 工程内。
  • Burst 默认使用 EPWM1 SOCA；DMA 默认走 EDMA1 CH0 与 DMAMUX ADC 请求源。
